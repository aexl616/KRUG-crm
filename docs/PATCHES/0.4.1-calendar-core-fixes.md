# Patch 0.4.1 — Calendar Core Fixes

**Тип:** Fix + UX

## Проблема

Patch 0.4 добавил ядро календаря и записей, но нужно стабилизировать мелкие проблемы перед переходом к 0.5.

---

## Почему сейчас

Без стабильного ядра календаря нельзя переходить к полировке, ролям, комнатам, Telegram и аналитике.

---

## Что изменить

1. Заменить старое название Studio Income на KRUG CRM / КРУГ CRM:
   - title страницы;
   - экран входа;
   - старые тексты, если они звучат неактуально.

2. Обновить клиентские подсказки:
   - клиенты должны собираться из `state.clients`;
   - `state.bookings`;
   - `state.payments`;
   - не только из платежей.

3. Удаление завершённой записи:
   - если у записи есть `paymentId` или связанный `payment.bookingId`;
   - показать предупреждение;
   - не удалять платёж автоматически;
   - не терять финансовые данные.

4. Улучшить читаемость календаря:
   - выбранная запись явно подсвечена;
   - пустой слот визуально кликабельный;
   - карточка показывает длительность;
   - карточка показывает цвет сотрудника;
   - статус хорошо читается.

5. Проверить отсутствие дублей платежей:
   - завершение записи создаёт платёж;
   - повторное завершение не создаёт дубль;
   - изменение суммы завершённой записи обновляет связанный платёж.

6. Добавить публичную полировку:
   - «Тестовая версия» или «Preview» в интерфейс;
   - экран входа выглядит как KRUG CRM, а не старый Studio Income.

---

## Что НЕ делать

- не добавлять роли;
- не добавлять drag & drop;
- не добавлять комнаты;
- не добавлять Telegram;
- не добавлять онлайн-запись;
- не делать большой редизайн;
- не менять стек;
- не делать масштабный рефакторинг.

---

## Проверка после выполнения

Проверить вручную:

- title страницы;
- создание записи;
- клиент из записи появляется в клиентах;
- клиент из записи появляется в подсказках;
- завершение записи создаёт платёж;
- повторное завершение не создаёт дубль;
- редактирование завершённой записи обновляет платёж;
- удаление завершённой записи не удаляет финансовые данные;
- календарь читаемый;
- CRM не содержит публичной клиентской формы.
